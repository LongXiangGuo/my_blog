>从swift编译的中间代码深入理解struct和class的区别

## 一、Swift 编译流程全景图

```
.swift 源文件
    ↓
[1. 解析 Parser] → 抽象语法树 (AST)
    ↓
[2. 语义分析 Sema] → 类型检查、派生意念
    ↓
[3. SIL 生成] → 原始 SIL (Raw SIL)
    ↓
[4. SIL 优化] → 规范化 SIL (Canonical SIL)  ← 我们重点看这层
    ↓
[5. IR 生成] → LLVM IR
    ↓
[6. 后端优化 + 代码生成] → 机器码 (.o)
    ↓
[7. 链接] → 可执行文件
```

**关键命令对应**：
| 步骤 | 命令 | 产物 |
|------|------|------|
| 查看 AST | `swiftc -dump-ast main.swift` | 语法树文本 |
| 查看 SIL | `swiftc -emit-sil main.swift` | **Raw SIL** |
| 查看优化后 SIL | `swiftc -emit-sil -O main.swift` | **Canonical SIL** |
| 查看 LLVM IR | `swiftc -emit-ir main.swift` | 中间表示 |
| 查看汇编 | `swiftc -emit-assembly main.swift` | 汇编代码 |
| 只编译不链接 | `swiftc -c main.swift` | `main.o` 目标文件 |

## 二、实战：对比 `struct` 与 `class` 的 SIL

### 2.1 创建一个示例文件 `point.swift`

```swift
// point.swift
struct PointStruct {
    var x: Int
    var y: Int
}

class PointClass {
    var x: Int
    var y: Int
    
    init(x: Int, y: Int) {
        self.x = x
        self.y = y
    }
}

// 使用示例
func testStruct() {
    var s1 = PointStruct(x: 10, y: 20)
    var s2 = s1          // 赋值
    s2.x = 99
}

func testClass() {
    let c1 = PointClass(x: 10, y: 20)
    let c2 = c1          // 赋值
    c2.x = 99
}
```

### 2.2 生成 SIL 并对比

```bash
swiftc -emit-sil -O point.swift > point.sil
```

让我们打开 `point.sil`，聚焦最关键的差异部分。

## 三、关键差异：SIL 代码对比

### 3.1 `struct` 的 SIL（简化标注版）

```swift
// struct PointStruct 的内存布局
// ① 定义：连续的两个 Int 字段，直接存储
sil_vtable PointStruct {}  // 注意：struct 没有 vtable（虚函数表）

// ② testStruct 函数的 SIL
sil hidden [ossa] @$s5point11testStructyyF : $@convention(thin) () -> () {
bb0:
    // 步骤1：在栈上分配 16 字节（两个 Int）
    %0 = alloc_stack $PointStruct, var, name "s1"
    
    // 步骤2：创建点 (10, 20) —— 直接写栈
    %1 = integer_literal $Builtin.Int64, 10
    %2 = integer_literal $Builtin.Int64, 20
    %3 = struct $PointStruct (%1 : $Builtin.Int64, %2 : $Builtin.Int64)
    store %3 to [trivial] %0
    
    // 步骤3：‼️ 关键：复制操作 —— 直接拷贝栈内存
    %4 = alloc_stack $PointStruct, var, name "s2"
    %5 = load [trivial] %0 : $*PointStruct   // 加载 s1
    store %5 to [trivial] %4 : $*PointStruct  // 存储到 s2（复制完成）
    
    // 步骤4：修改 s2.x —— 在拷贝的栈上修改
    %6 = load [trivial] %4 : $*PointStruct
    %7 = struct_extract %6 : $PointStruct, #PointStruct.x  // 取 x
    %8 = integer_literal $Builtin.Int64, 99
    %9 = struct_element_addr %4 : $*PointStruct, #PointStruct.x
    assign %8 to %9 : $*Builtin.Int64
    
    // 步骤5：释放栈内存
    dealloc_stack %4
    dealloc_stack %0
    %10 = tuple ()
    return %10
}
```

**`struct` 关键特征**：
- ❌ **没有 vtable**（无虚表，无动态派发）
- ✅ **所有操作在栈上**（`alloc_stack`、`store`、`load`）
- ✅ **赋值 = 内存拷贝**（`load` + `store`）
- ✅ **值语义**：修改 `s2` 不会影响 `s1`

### 3.2 `class` 的 SIL（简化标注版）

```swift
// class PointClass 的内存布局
// ① 类元数据 + 引用计数头部 + 两个 Int 字段
sil_vtable PointClass {
  #PointClass.init!allocator: (PointClass.Type) -> (Int, Int) -> PointClass : @$s5point11PointClassC1x1yACSi_SitcfC
  #PointClass.deinit!deallocator: @$s5point11PointClassCfD
}

// ② testClass 函数的 SIL
sil hidden [ossa] @$s5point11testClassyyF : $@convention(thin) () -> () {
bb0:
    // 步骤1：在堆上分配对象（返回指针）
    %0 = alloc_ref $PointClass
    
    // 步骤2：初始化对象（x=10, y=20）
    %1 = integer_literal $Builtin.Int64, 10
    %2 = integer_literal $Builtin.Int64, 20
    // 初始化引用计数 + 字段赋值
    %3 = class_method %0 : $PointClass, #PointClass.init!allocator.1
    %4 = apply %3(%1, %2, %0) : $@convention(method) (Int, Int, @owned PointClass) -> @owned PointClass
    
    // 步骤3：‼️ 关键：复制操作 —— 只拷贝指针（引用计数 +1）
    strong_retain %4 : $PointClass   // 增加引用计数
    %5 = copy_value %4 : $PointClass  // 拷贝引用（不是对象本身）
    // 此时 %4 和 %5 指向同一块堆内存
    
    // 步骤4：修改 c2.x —— 通过指针修改堆上的值
    %6 = ref_element_addr %5 : $PointClass, #PointClass.x  // 获取字段地址
    %7 = integer_literal $Builtin.Int64, 99
    assign %7 to %6 : $*Builtin.Int64
    
    // 步骤5：释放引用（引用计数 -1）
    strong_release %5 : $PointClass
    strong_release %4 : $PointClass
    
    %8 = tuple ()
    return %8
}
```

**`class` 关键特征**：
- ✅ **有 vtable**（支持方法动态派发和继承）
- ✅ **堆分配**（`alloc_ref`）
- ✅ **引用计数**（`strong_retain`/`strong_release`）
- ✅ **赋值 = 拷贝指针**（`copy_value` 只复制引用）
- ✅ **引用语义**：修改 `c2.x` 会同时影响 `c1`

## 四、核心指令对照表

| 操作 | `struct` (SIL 指令) | `class` (SIL 指令) |
|------|---------------------|--------------------|
| **内存分配** | `alloc_stack` (栈) | `alloc_ref` (堆) |
| **存储值** | `store [trivial]` | `store` (存指针) |
| **加载值** | `load [trivial]` | `load` (加载指针) |
| **复制** | 内存拷贝 (`load`+`store`) | `copy_value` (仅拷贝指针) |
| **字段访问** | `struct_element_addr` (直接偏移) | `ref_element_addr` (通过引用) |
| **方法调用** | 静态派发 (`struct` 无 vtable) | `class_method` (查 vtable) |
| **生命周期** | 作用域结束自动 `dealloc_stack` | `strong_release` (ARC) |

## 五、更深入的对比：传递行为在底层

回到你之前的疑问 **"传递时会自动拷贝，地址会不会变"**，现在可以从 SIL 层面得到印证：

### `struct` 的传递（值拷贝）
```swift
func takeStruct(_ p: PointStruct) {
    var local = p   // ‼️ SIL 中对应：load p 的栈内存，store 到 local 的栈
    local.x = 99
}
```
**SIL 表现**：完整的 `load` + `store` → **内存拷贝**。

### `class` 的传递（引用拷贝）
```swift
func takeClass(_ c: PointClass) {
    let local = c   // ‼️ SIL 中对应：strong_retain c，copy_value c
    local.x = 99
}
```
**SIL 表现**：只拷贝**指针**（8 字节），引用计数 +1。

## 六、如何自己动手验证

### 6.1 完整操作流程

```bash
# 1. 创建 Swift 文件
cat > demo.swift << 'EOF'
struct S { var value: Int }
class C { var value: Int; init(value: Int) { self.value = value } }

func test() {
    var s1 = S(value: 10)
    var s2 = s1
    s2.value = 99
    
    let c1 = C(value: 10)
    let c2 = c1
    c2.value = 99
}
EOF

# 2. 生成可读的 SIL
swiftc -emit-sil -O demo.swift > demo.sil

# 3. 查看关键部分（结构体）
sed -n '/struct S/,/^$/p' demo.sil

# 4. 查看关键部分（类）
sed -n '/class C/,/^$/p' demo.sil

# 5. 生成 LLVM IR（更底层，可选）
swiftc -emit-ir -O demo.swift > demo.ll
```

### 6.2 关键过滤命令

```bash
# 只看 test 函数的 SIL
swiftc -emit-sil -O demo.swift | grep -A 50 "test()"

# 查看引用计数操作
swiftc -emit-sil -O demo.swift | grep -E "retain|release|copy_value"

# 查看内存分配
swiftc -emit-sil -O demo.swift | grep -E "alloc_stack|alloc_ref"
```

## 七、核心总结

| 层面 | `struct` | `class` |
|------|----------|---------|
| **SIL 指令数量** | 少（栈操作） | 多（堆分配+引用计数） |
| **内存操作** | `alloc_stack` / `dealloc_stack` | `alloc_ref` / `strong_release` |
| **赋值语义** | 内存拷贝 | 指针拷贝 + 引用计数 |
| **方法派发** | 静态（编译时确定） | 动态（通过 vtable） |
| **线程安全** | 天然安全（无共享） | 需额外同步 |

**一句话总结 SIL 层的区别**：
> `struct` 在 SIL 中是连续的内存块，赋值就是 `memcpy`；`class` 是带引用计数头的堆对象，赋值就是拷贝指针并 `retain`。

你现在手里有终端吗？如果方便的话，可以直接跑一下上面的命令，看看你自己的代码生成的 SIL，会非常有冲击力——**Swift 编译器其实把 `struct` 和 `class` 的差异翻译得极其清晰直观**。
