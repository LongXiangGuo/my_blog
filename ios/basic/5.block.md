# Block
优缺点

优点	说明
代码连贯性	回调逻辑紧邻调用处，提高可读性
闭包特性	自动捕获上下文变量，减少参数传递
类型安全	编译时检查参数和返回值类型
灵活性	可作为参数、返回值、存入集合
性能	比 NSInvocation 快，接近函数指针

❌ 缺点
缺点	说明	解决方案
循环引用	捕获 self 导致内存泄漏	使用 __weak/__unsafe_unretained

栈Block限制	栈Block离开作用域失效	使用 copy 移到堆上
类型冗长	声明复杂	使用 typedef

## 三种类型

```c
// 1. 全局 Block (NSGlobalBlock)
void (^globalBlock)(void) = ^{
    NSLog(@"Global Block");
};
// 不捕获外部变量，存储在 __TEXT 段（代码段）

// 2. 栈 Block (NSStackBlock)
int value = 10;
void (^stackBlock)(void) = ^{
    NSLog(@"%d", value);  // 捕获外部变量
};
// MRC 下存储在栈，ARC 下会自动拷贝

// 3. 堆 Block (NSMallocBlock)
void (^heapBlock)(void) = [^{
    NSLog(@"Heap Block");
} copy];
// 存储在堆，需要手动/自动管理内存
```

## 性能优化

- 使用weakSelf防止循环引用

```Objective-C
__weak typeof(self) weakSelf = self;
self.block = ^{
    __strong typeof(weakSelf) strongSelf = weakSelf;
};
```

- 避免在循环中处理block逻辑，单独处理

- 修改外部变量

```c
// ❌ 错误：不能修改值捕获的变量
int count = 0;
void (^block)(void) = ^{
    count++;  // 编译错误！
};

// ✅ 解决方案1：使用 __block
__block int count = 0;

NSMutableArray *array = [NSMutableArray arrayWithObject:@0];

    array[0] = @([array[0] intValue] + 1);
};
```

## Questions

__block 修饰的是指针变量本身（即 array 这个变量），而不是指针指向的对象。当你修改数组内容时，你改变的是对象内部的状态，而不是 array 变量指向哪个对象。

这是一个非常核心的问题！Block 与三区的关系直接决定了它的生命周期和行为。

## 一、三种 Block 与内存区域的关系

```objectivec
// 1. 全局区 Block (NSGlobalBlock)
void (^globalBlock)(void) = ^{
    NSLog(@"Global Block");
};

// 2. 栈区 Block (NSStackBlock) - MRC 下明显，ARC 下会自动 copy
int value = 10;
void (^stackBlock)(void) = ^{
    NSLog(@"%d", value);
};

// 3. 堆区 Block (NSMallocBlock)
void (^heapBlock)(void) = [^{
    NSLog(@"Heap Block");
} copy];
```

### 对应关系表

| Block 类型 | 存储区域 | 生命周期 | 捕获外部变量 | 可修改性 |
|-----------|---------|---------|-------------|---------|
| **全局 Block** | 全局区（数据段） | 程序启动到结束 | 不能捕获（或只捕获静态/全局变量） | 只读 |
| **栈 Block** | 栈区 | 函数执行期间 | 捕获自动变量 | 离开作用域销毁 |
| **堆 Block** | 堆区 | 手动/自动管理 | 拷贝所有捕获变量 | 引用计数管理 |

## 二、详细分析

### 1. **全局区 Block**

```objectivec
// 情况1：不捕获任何外部变量
void (^block1)(void) = ^{
    NSLog(@"Hello");
};  // → 全局 Block

// 情况2：只捕获全局变量或静态变量
static int globalInt = 10;
void (^block2)(void) = ^{
    NSLog(@"%d", globalInt);
};  // → 全局 Block

// 内存布局
// ┌─────────────────┐ 高地址
// │      栈区       │
// ├─────────────────┤
// │      堆区       │
// ├─────────────────┤
// │   全局 Block    │ ← __TEXT, __DATA 段
// │  (程序生命周期)  │
// ├─────────────────┤
// │      代码段     │
// └─────────────────┘ 低地址
```

### 2. **栈区 Block**

```objectivec
// MRC 环境下的典型栈 Block
- (void)testStackBlock {
    int local = 10;  // 栈上的局部变量
    
    // 这个 Block 创建在栈上
    void (^stackBlock)(void) = ^{
        NSLog(@"%d", local);
    };
    
    // ⚠️ 危险！返回栈 Block 会导致悬垂指针
    return stackBlock;  // 函数返回后，栈内存被回收
}
```

**栈 Block 的结构**：
```
栈区增长方向 → 低地址向高地址
┌─────────────┐
│ 局部变量    │
├─────────────┤
│ 参数        │
├─────────────┤
│ Block 结构  │ ← isa: _NSConcreteStackBlock
│  - isa      │   flags: BLOCK_IS_STACK
│  - flags    │   invoke: 实现代码
│  - invoke   │   捕获的 local = 10（值拷贝）
│  - captured │
└─────────────┘
```

### 3. **堆区 Block**

```objectivec
// 堆 Block 的创建
- (void)createHeapBlock {
    int local = 10;
    
    // 方式1：手动 copy
    void (^heapBlock1)(void) = [^{
        NSLog(@"%d", local);
    } copy];
    
    // 方式2：ARC 下自动 copy
    void (^heapBlock2)(void) = ^{
        NSLog(@"%d", local);
    };  // ARC 会自动 copy 到堆上
    
    // 方式3：作为返回值（ARC 自动）
    return ^{
        NSLog(@"%d", local);
    };
}
```

**堆 Block 的结构**：
```
堆区
┌─────────────────────┐
│ Block_layout        │ ← isa: _NSConcreteMallocBlock
│  - isa              │   flags: BLOCK_IS_COPIED
│  - flags            │   引用计数 = 1
│  - invoke           │
│  - descriptor       │
│  - captured_value   │
├─────────────────────┤
│ __block 变量结构体   │ ← 如果有 __block 变量
└─────────────────────┘
```

## 三、内存转移过程

### **栈 → 堆 的拷贝过程**

```objectivec
// 原始栈 Block
struct Block_layout stackBlock = {
    .isa = _NSConcreteStackBlock,
    .flags = 0,
    .invoke = __test_block_invoke,
    .captured = {value: 10}
};

// 执行 copy 操作
void *_Block_copy(void *arg) {
    struct Block_layout *stackBlock = arg;
    
    // 1. 在堆上分配内存
    struct Block_layout *heapBlock = malloc(blockSize);
    
    // 2. 拷贝数据
    memcpy(heapBlock, stackBlock, blockSize);
    
    // 3. 修改 isa 为堆 Block
    heapBlock->isa = _NSConcreteMallocBlock;
    
    // 4. 处理捕获的对象
    if (flags & BLOCK_HAS_COPY_DISPOSE) {
        // 调用 copy helper，对捕获的对象发送 retain
        descriptor->copy(heapBlock, stackBlock);
    }
    
    return heapBlock;
}
```

### **拷贝前后对比**

```objectivec
// 拷贝前
__block int value = 10;
void (^stackBlock)(void) = ^{
    value++;
};

// 栈上结构：
// stackBlock (栈) ──捕获──> __block_byref_value (栈)
//                              └─ forwarding 指向自己

// 拷贝后（调用 [stackBlock copy]）
// stackBlock (栈) ──拷贝──> heapBlock (堆)
//                              │
//                              ▼
//                    __block_byref_value (堆)
//                    栈上的 forwarding ─┘指向堆版本
```

## 四、ARC 下的自动优化

```objectivec
// ARC 会自动将栈 Block copy 到堆上的常见场景

// 1. 赋值给强引用变量
@property (strong) void (^myBlock)(void);
self.myBlock = ^{
    NSLog(@"Auto copy");
};  // 自动 copy 到堆

// 2. 作为方法参数（多数情况）
[array enumerateObjectsUsingBlock:^(id obj, NSUInteger idx, BOOL *stop) {
    // 系统会自动处理 copy
}];

// 3. 作为返回值
- (void (^)(void))getBlock {
    int value = 10;
    return ^{
        NSLog(@"%d", value);
    };  // ARC 自动 copy 到堆
}

// 4. 赋值给 __strong 变量
void (^strongBlock)(void) = ^{
    // 自动堆 Block
};
```

## 五、实际内存地址验证

```objectivec
#import <objc/runtime.h>

void testBlockMemory() {
    // 1. 全局 Block
    void (^global)(void) = ^{ NSLog(@"Global"); };
    NSLog(@"Global: %p, 类型: %@", global, [global class]);
    // 输出: Global: 0x10xxxx, 类型: __NSGlobalBlock__
    
    // 2. 栈 Block (MRC 下验证)
    int local = 10;
    void (^stack)(void) = ^{ NSLog(@"%d", local); };
    #if !__has_feature(objc_arc)
    NSLog(@"Stack: %p, 类型: %@", stack, [stack class]);
    // 输出: Stack: 0x7ffee..., 类型: __NSStackBlock__
    #endif
    
    // 3. 堆 Block
    void (^heap)(void) = [stack copy];
    NSLog(@"Heap: %p, 类型: %@", heap, [heap class]);
    // 输出: Heap: 0x6000..., 类型: __NSMallocBlock__
    
    [heap release];  // MRC 需要手动释放
}

// 验证地址范围
// 0x1xxxxxxx - 0x4xxxxxxx → 代码段/全局区
// 0x7xxxxxxx → 栈区
// 0x6xxxxxxx → 堆区
```

## 六、常见陷阱与最佳实践

### **陷阱1：返回栈 Block**
```objectivec
// ❌ 错误
typedef void (^MyBlock)(void);
- (MyBlock)badMethod {
    int value = 10;
    return ^{
        NSLog(@"%d", value);
    };  // MRC 下返回栈 Block，悬垂指针
}

// ✅ 正确
- (MyBlock)goodMethod {
    int value = 10;
    return [^{
        NSLog(@"%d", value);
    } copy];  // 主动 copy 到堆
    // ARC 下自动处理
}
```

### **陷阱2：Block 生命周期管理**
```objectivec
// ❌ 错误：栈 Block 过早释放
void (^block)(void);
if (condition) {
    int temp = 100;
    block = ^{
        NSLog(@"%d", temp);
    };
    // 这里 block 在栈上，离开 if 作用域就销毁
}
block();  // ⚠️ 野指针崩溃

// ✅ 正确：使用 copy
if (condition) {
    int temp = 100;
    block = [^{
        NSLog(@"%d", temp);
    } copy];
}
```

## 七、总结对比表

| 内存区域 | Block 类型 | 创建时机 | 销毁时机 | 捕获变量 | 需要 copy |
|---------|-----------|---------|---------|---------|----------|
| **全局区** | `__NSGlobalBlock__` | 编译时 | 程序结束 | 无/静态变量 | ❌ |
| **栈区** | `__NSStackBlock__` | 运行时（函数调用） | 离开作用域 | 自动变量 | ✅ 需要 |
| **堆区** | `__NSMallocBlock__` | copy 操作 | 引用计数为0 | 拷贝到堆 | ✅ 已完成 |

**记忆要点**：
1. **不捕获外部变量** → 全局区（永恒）
2. **捕获且未 copy** → 栈区（临时）
3. **捕获且已 copy** → 堆区（可控）

**ARC 最佳实践**：
- 绝大多数情况下，ARC 自动管理，直接写即可
- 作为属性用 `copy` 修饰
- 避免在 MRC 中手动管理栈 Block
- 注意循环引用（用 `__weak`）

## 悬垂指针（Dangling Pointer）
悬垂指针是指向已经被释放或无效内存地址的指针。访问它会导致未定义行为（通常是崩溃）。

## Questions

1. block使用copy后 会自动搬运到堆上么?

对 Block 发送 copy 消息后，它会被自动搬运（拷贝）到堆上。

2. block->isa == &_NSConcreteMallocBlock) 怎么理解 &_NSConcreteMallocBlock

```C
// 在 Objective-C 运行时中，有3个全局符号（全局变量）
extern void * _NSConcreteGlobalBlock;   // 全局 Block 的类
extern void * _NSConcreteStackBlock;    // 栈 Block 的类
extern void * _NSConcreteMallocBlock;   // 堆 Block 的类



// 运行时实际的行为（伪代码）
// 在 dyld 加载时，这些符号被特殊初始化
void * _NSConcreteMallocBlock = (void*)&_NSConcreteMallocBlock;

// 验证：
printf("%p\n", &_NSConcreteMallocBlock);  // 0x100001000
printf("%p\n", _NSConcreteMallocBlock);   // 0x100001000（相同！）
```

>关键点：_NSConcreteMallocBlock 在编译器和运行时的定义是特殊处理的，它的值被设计成等于它自己的地址！

## 内存模型

```objectivec
// 这些符号在 __DATA 段的布局
// 地址：0x100008000  _NSConcreteGlobalBlock:
// 地址：0x100008008  _NSConcreteStackBlock:

_NSConcreteGlobalBlock  = 0x100008000

&_NSConcreteGlobalBlock  = 0x100008000
```
