# Swift SIL 完整解析文档：struct vs class

## 一、文件头部信息

```swift
sil_stage canonical
```
**含义**：声明 SIL 的优化阶段为 `canonical`（规范化阶段）。说明这份 SIL 已经过基本优化（如泛型特化、AR C优化等），但不是最终优化后的版本（如 `-Osize` 会更激进）。

```swift
import Builtin
import Swift
import SwiftShims
```
- **Builtin**：Swift 编译器内置模块，提供对底层语言特性的直接访问（如 `integer_literal`、`alloc_ref` 等），开发者通常无法直接使用。
- **Swift**：Swift 标准库。
- **SwiftShims**：与 C/Obj-C 互操作时的桥接辅助模块。

---

## 二、类型声明（原始源码的结构体/类定义）

```swift
struct S {
  @_hasStorage var value: Int { get set }
  init(value: Int)
}
```
- **`@_hasStorage`**：编译器内部标记，表示该属性有实际的存储空间（而不是计算属性）。
- 结构体 `S` 包含一个可读写的存储属性 `value`，和一个指定初始化器。

```swift
class C {
  @_hasStorage var value: Int { get set }
  init(value: Int)
  @objc deinit
}
```
- 类 `C` 拥有与 `S` 类似的属性，但额外声明了 `@objc deinit`（表示析构函数对 Objective-C 运行时可见，用于兼容性）。

```swift
func test()
```
- 函数声明（没有具体实现体，因为在 `-O` 优化下内部操作被优化掉了，只剩下函数签名）。

---

## 三、程序入口点 `main`

```swift
sil @main : $@convention(c) (Int32, UnsafeMutablePointer<Optional<UnsafeMutablePointer<Int8>>>) -> Int32
```
- **`@main`**：标注该函数为程序入口点（类似于 C 的 `main` 函数）。
- **`$@convention(c)`**：调用约定为 C 语言调用约定，以便与操作系统交互。
- **参数**：
  - `argc`（`Int32`）：命令行参数个数。
  - `argv`（复杂指针类型）：命令行参数数组。
- **返回值**：`Int32`，返回 0 表示正常退出。

```swift
bb0(%0 : $Int32, %1 : $UnsafeMutablePointer<Optional<UnsafeMutablePointer<Int8>>>):
```
- **基本块（Basic Block）** `bb0`：函数入口。
- `%0`、`%1`：SIL 寄存器，分别接收 `argc` 和 `argv`。

```swift
  %2 = integer_literal $Builtin.Int32, 0
```
- 创建 `Builtin.Int32` 类型的字面量 `0`。

```swift
  %3 = struct $Int32 (%2)
```
- 将内建类型 `Builtin.Int32` 包装成 Swift 标准库的 `struct Int32`。

```swift
  return %3
```
- 返回 `%3`（即 `0`）。

**总结**：`main` 函数只是简单返回 0，因为真正的程序逻辑由 Swift 运行时初始化，并且 `test` 函数在优化后没有生成具体操作。

---

## 四、结构体 `S` 的存取方法

### 4.1 getter（读取 value）

```swift
sil hidden [transparent] @$s4demo1SV5valueSivg : $@convention(method) (S) -> Int
```
- **`[transparent]`**：该函数被标记为“透明的”，编译器在调用处可能会内联展开。
- **`@$s4demo...`**：被修饰后的函数符号名（name mangling）。
- **参数**：接收一个 `S` 实例（值类型，直接传递）。
- **返回值**：`Int`。

```swift
bb0(%0 : $S):
```
- `%0`：传入的结构体实例本身（**值**）。

```swift
  debug_value %0, let, name "self", argno 1
```
- **调试信息**：标记 `%0` 为常量 `self`（用于调试器）。

```swift
  %2 = struct_extract %0, #S.value
```
- **`struct_extract`**：从结构体值 `%0` 中提取字段 `#S.value`。
- 对于值类型，字段的访问就是直接内存偏移量提取，开销极低。

```swift
  return %2
```
- 返回提取的 `Int` 值。

### 4.2 setter（修改 value）

```swift
sil hidden [transparent] @$s4demo1SV5valueSivs : $@convention(method) (Int, @inout S) -> ()
```
- **参数**：
  - `Int`：要设置的新值。
  - `@inout S`：结构体实例的**地址**（因为需要修改其内容）。

```swift
bb0(%0 : $Int, %1 : $*S):
```
- `%0`：新值。
- `%1`：指向结构体 `S` 的指针（栈上地址）。

```swift
  %4 = struct $S (%0)
```
- 用新值 `%0` 创建一个**全新的结构体实例**。

```swift
  store %4 to %1
```
- 将新创建的结构体写入 `%1` 指向的内存位置（覆盖旧值）。

**关键观察**：修改 `struct` 的字段时，实际上是**重新构造一个完整的新结构体**并覆盖原内存。这就是值语义的实现方式。

## 五、类 `C` 的存取方法

### 5.1 getter

```swift
sil hidden [transparent] @$s4demo1CC5valueSivg : $@convention(method) (@guaranteed C) -> Int
```
- **`@guaranteed`**：参数是引用类型，调用方保证该对象在函数执行期间存活，函数不需要额外 retain。
- 参数：`C` 类型的实例引用。

```swift
bb0(%0 : $C):
  %2 = ref_element_addr %0, #C.value
```
- **`ref_element_addr`**：获取引用类型 `%0` 中字段 `#C.value` 的**内存地址**（堆上的偏移量）。

```swift
  %3 = begin_access [read] [dynamic] [no_nested_conflict] %2
```
- **`begin_access`**：开启内存访问（Swift 的运行时排他性检查）。`[read]` 表示读访问。

```swift
  %4 = load %3
```
- 从该地址加载值（`Int`）。

```swift
  end_access %3
```
- 结束访问。

```swift
  return %4
```

**区别点**：类需要通过 `ref_element_addr` + 内存访问来读取字段，而结构体直接用 `struct_extract`。

### 5.2 setter

```swift
sil hidden [transparent] @$s4demo1CC5valueSivs : $@convention(method) (Int, @guaranteed C) -> ()
```
- 参数：新值 `Int`，类实例 `C`。

```swift
  %4 = ref_element_addr %1, #C.value
  %5 = begin_access [modify] [dynamic] [no_nested_conflict] %4
```
- **`[modify]`**：表示将要修改该内存。

```swift
  store %0 to %5
```
- 将新值存储到该地址。

**区别点**：类修改字段时不需要重新构造整个对象，只是修改堆上的一个内存位置。

## 六、初始化器对比

### 结构体初始化器

```swift
sil hidden @$s4demo1SV5valueACSi_tcfC : $@convention(method) (Int, @thin S.Type) -> S
```
- **`@thin`**：类型 `S.Type` 在编译时已知，没有运行时元数据开销。
- 参数：`Int`（value）、元类型（`S.Type`）。
- 返回值：`S` 实例（**值类型**）。

```swift
bb0(%0 : $Int, %1 : $@thin S.Type):
  %2 = struct $S (%0)
  return %2
```
- 直接用字面量构建结构体并返回（值，在栈上）。

### 类初始化器

```swift
sil hidden [exact_self_class] @$s4demo1CC5valueACSi_tcfC : $@convention(method) (Int, @thick C.Type) -> @owned C
```
- **`@thick`**：类类型需要运行时元数据（因为有继承、vtable 等）。
- **`@owned`**：返回的对象由调用方负责释放（ARC 的 +1）。
- `[exact_self_class]`：保证返回的对象的动态类型等于 `C` 本身（不是子类）。

```swift
  %2 = alloc_ref $C
```
- **`alloc_ref`**：在**堆**上分配一个 `C` 对象（包括引用计数头部空间）。

```swift
  %5 = end_init_let_ref %2
```
- 标记对象初始化完成（引用计数变为 1）。

```swift
  %6 = ref_element_addr %5, #C.value
  store %0 to %6
```
- 存储字段值到堆内存。

```swift
  return %5
```
- 返回堆对象的引用（指针）。

**区别点**：结构体用 `struct $S` 在栈上构建，类用 `alloc_ref` 在堆上分配。

---

## 七、析构函数（deinit）

```swift
sil hidden @$s4demo1CCfd : $@convention(method) (@guaranteed C) -> @owned Builtin.NativeObject
```
- `C.fd`：`fd` 代表 `finish deinit`，即将调用用户的 `deinit` 方法。
- 返回 `Builtin.NativeObject` 以便释放内存。

```swift
sil hidden @$s4demo1CCfD : $@convention(method) (@owned C) -> ()
```
- `C.fD`：`fD` 代表 `final deinit`，执行真正的内存释放（`dealloc_ref`）。

---

## 八、函数 `test()`（已被优化）

```swift
sil hidden @$s4demo4testyyF : $@convention(thin) () -> ()
```
- 函数没有参数，没有返回值。

```swift
bb0:

```
- **`undef`**：未定义值（表示这些变量在优化后被消除）。
- `debug_value` 仅保留用于调试信息（变量名、类型、字面量值 99、10 等），但没有生成实际内存操作。

```swift
  %3 = tuple ()
  return %3
```
- 返回空元组 `Void`。

**说明**：由于 `test()` 内的操作没有外部可见副作用，`-O` 优化将所有内存操作删除了，只保留了调试信息。

## 九、虚函数表（vtable）

```swift
sil_vtable C {
  #C.value!getter: ... : @$s4demo1CC5valueSivg
  #C.value!setter: ... : @$s4demo1CC5valueSivs
  #C.value!modify: ... : @$s4demo1CC5valueSivM
  #C.init!allocator: ... : @$s4demo1CC5valueACSi_tcfC
  #C.deinit!deallocator: ... : @$s4demo1CCfD
}
```
- **vtable**：类的**虚函数表**（仅类拥有，结构体没有）。
- 用于支持动态派发（多态）、方法重写、继承。
- 每个类方法对应一个入口，包含函数指针。

**重要观察**：`sil_vtable S` 不存在。

## 十、核心区别对比总结

| 方面 | 结构体 `S` | 类 `C` |
|------|------------|--------|
| **存储位置** | 栈（按值传递） | 堆（通过引用传递） |
| **分配指令** | 无显式分配（函数帧内直接使用） | `alloc_ref` |
| **复制语义** | 通过 `struct $S` 重建 | 通过 `copy_value`/`strong_retain` 增加引用计数 |
| **字段访问** | `struct_extract`（直接偏移） | `ref_element_addr`（通过引用） + 内存访问 |
| **修改字段** | 重建整个结构体并覆盖 | 直接修改堆上的内存 |
| **初始化** | 直接返回栈上的值 | 返回堆对象的引用（`@owned`） |
| **生命周期** | 离开作用域自动释放（栈展开） | 通过 ARC 管理（`strong_release`） |
| **vtable** | ❌ 不存在 | ✅ 存在，支持动态派发 |
| **方法调用** | 静态派发 | 动态派发（通过 vtable） |
| **线程安全性** | 天然安全（独立副本） | 需额外同步（共享状态） |
