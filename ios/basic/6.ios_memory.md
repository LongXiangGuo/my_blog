## 二、详细内存分类

### 1. **代码段 (__TEXT)**

```objectivec
// 存储在代码段的内容
// ├─ 机器指令
// ├─ 常量字符串
// └─ 只读全局常量

const int readOnlyGlobal = 100;          // 只读数据段
NSString *constString = @"Hello";         // 常量字符串（__cstring）
void (^globalBlock)(void) = ^{            // 全局 Block
    NSLog(@"Global");
};

// 特点：
// ✅ 只读，不可修改
// ✅ 程序启动时加载，程序结束时释放
// ✅ 所有线程共享
```

### 2. **数据段 (__DATA)**

```objectivec
// 可读写数据
int globalVar = 10;                        // 已初始化全局变量（__data）
static int staticVar = 20;                 // 静态变量（__data）
int uninitGlobal;                          // 未初始化全局变量（__bss）
static int uninitStatic;                   // 未初始化静态变量（__bss）

// __data：已初始化数据
// __bss：未初始化数据（程序启动时清零）

// 特点：
// ✅ 可读写
// ✅ 程序启动到结束
// ✅ 线程共享需同步
```

- **__bss**: 存放未初始化的全局变量、静态变量，不占磁盘文件大小，但占运行时内存。
未初始化的全局变量	int g_counter;	✅
未初始化的静态变量	static int s_value;	✅
初始化为 0 的全局变量	int g_flag = 0;	✅（多数情况）
**典型表现：**
你定义一个很大的 static char buffer[1024*1024];
可执行文件体积几乎不变，但 App 跑起来会多占 1MB 内存

- **__objc_data** 是什么（OC 运行时核心）
一句话定义：
存放 Objective‑C 类在编译期生成的运行时元数据（类结构、方法列表、协议、属性等）

它属于 __DATA 段，是 可读可写 的。

里面具体存什么？
内容	说明
类描述	struct class_t
方法列表	method_list_t
协议列表	protocol_list_t
属性列表	property_list_t
类名、实例大小等	运行时必要的信息
👉 每一个你在代码里写的 @interface … @end，最终都在 __objc_data 里有对应的元数据。

### 3. **堆区 (Heap)**

```objectivec
// 动态分配的内存
NSObject *obj = [[NSObject alloc] init];   // OC 对象，大小 16+ 字节
int *buffer = malloc(1024);                 // C 缓冲区
void (^heapBlock)(void) = [^{              // 堆 Block
    int x = 10;
    NSLog(@"%d", x);
} copy];

// 内存管理方式：
// ├─ MRC：手动 retain/release/autorelease
// ├─ ARC：编译器自动插入 retain/release
// └─ 手动：malloc/free

// 特点：
// ⚠️ 需手动或自动管理生命周期
// ⚠️ 可能产生内存泄漏
// ✅ 大小灵活，可动态增长
// 📈 向高地址增长
```

### 4. **栈区 (Stack)**

```objectivec
void function(int param) {                 // 参数在栈上
    int localVar = 10;                     // 局部变量在栈上
    int array[100];                        // 数组在栈上
    void (^stackBlock)(void) = ^{          // 栈 Block（MRC）
        NSLog(@"%d", localVar);
    };
    
    // 特点：
    // ✅ 自动管理（函数返回时自动销毁）
    // ✅ 访问速度快
    // ⚠️ 大小有限（通常 8MB）
    // ⚠️ 不能返回栈变量的指针
    // 📉 向低地址增长
}
```

## 三、内存管理机制详解

### 1. **ARC (Automatic Reference Counting)**

```objectivec
// 编译器在编译时自动插入 retain/release
// ARC 前
NSObject *obj = [[NSObject alloc] init];
[obj retain];
[obj release];

// ARC 后（编译器自动插入）
NSObject *obj = [[NSObject alloc] init];
// 编译器插入：objc_retainAutoreleasedReturnValue(obj)
// 编译器插入：objc_release(obj) 在作用域结束时

// 引用计数管理规则
__strong NSObject *strongObj = [[NSObject alloc] init];  // 引用计数 +1
__weak NSObject *weakObj = strongObj;                     // 引用计数不变
__unsafe_unretained NSObject *unsafeObj = strongObj;     // 不管理，不安全
__autoreleasing NSObject *autoObj = [[NSObject alloc] init]; // 自动释放池
```

### 2. **引用计数存储方式**

```objectivec
// 方式1：extra retain count（存储在 isa 中）
// 64位下，isa 指针的部分位用于存储引用计数
union isa_t {
    uintptr_t bits;
    struct {
        uintptr_t nonpointer        : 1;  // 是否优化
        uintptr_t has_assoc         : 1;  // 有关联对象
        uintptr_t has_cxx_dtor      : 1;  // 有 C++ 析构
        uintptr_t shiftcls          : 33; // 类指针
        uintptr_t magic             : 6;  // 调试标识
        uintptr_t weakly_referenced : 1;  // 有弱引用
        uintptr_t deallocating      : 1;  // 正在释放
        uintptr_t has_sidetable_rc  : 1;  // 引用计数溢出到 SideTable
        uintptr_t extra_rc          : 19; // 额外的引用计数（实际引用计数 = extra_rc + 1）
    };
};

// 方式2：SideTable（当 extra_rc 溢出时）
struct SideTable {
    spinlock_t slock;
    RefcountMap refcnts;  // 存储溢出部分的引用计数
    weak_table_t weak_table;
};
```

### 3. **Autorelease Pool 机制**

```objectivec
// 自动释放池的工作原理
@autoreleasepool {
    // 1. 创建 AutoreleasePoolPage
    NSObject *obj = [[[NSObject alloc] init] autorelease];
    // 2. obj 被添加到当前页的栈中
    // 3. 作用域结束时，调用 objc_autoreleasePoolPop
    // 4. 遍历发送 release 消息
}

// AutoreleasePoolPage 结构
class AutoreleasePoolPage {
    magic_t const magic;
    id *next;                    // 下一个存放位置
    pthread_t const thread;      // 所属线程
    AutoreleasePoolPage *const parent;
    AutoreleasePoolPage *child;
    uint32_t const depth;
    uint32_t hiwat;
    
    // 存储 autorelease 对象
    id begin[];                  // 对象指针数组
};
```

### 4. **Weak 引用管理**

```objectivec
// weak 表的结构
// SideTable 中包含 weak_table_t
struct weak_table_t {
    weak_entry_t *weak_entries;  // 哈希表
    size_t num_entries;
    uintptr_t mask;
    uintptr_t max_hash_displacement;
};

// weak_entry_t 存储关系
struct weak_entry_t {
    DisguisedPtr<objc_object> referent;  // 被指向的对象
    union {
        struct {
            weak_referrer_t *referrers;   // 指向该对象的 weak 指针列表
            uintptr_t out_of_line : 1;
        };
        struct {
            weak_referrer_t inline_referrers[4];  // 内联数组（优化）
        };
    };
};

// 对象释放时
// 1. 从 weak 表找到所有指向它的 weak 指针
// 2. 将所有 weak 指针置为 nil
// 3. 从 weak 表移除记录
```

## 四、内存管理生命周期

### **OC 对象的完整生命周期**

```objectivec
// 1. 分配内存
NSObject *obj = [NSObject alloc];
// └→ calloc(size) 分配内存
// └→ isa 指针初始化

// 2. 初始化
obj = [obj init];
// └→ 可能返回不同的对象（如单例）

// 3. 使用期间
[obj retain];   // 引用计数 +1
[obj retain];   // 引用计数 +1
[obj release];  // 引用计数 -1

// 4. dealloc（引用计数为 0）
// ├→ 调用 dealloc
// ├→ 如果有 weak 引用，置 nil
// ├→ 移除关联对象
// ├→ 调用 C++ 析构
// └→ free() 释放内存
```

## 五、内存管理最佳实践

### 1. **避免循环引用**

```objectivec
// 闭包中的循环引用
@property (strong) void (^block)(void);

// ❌ 错误
self.block = ^{
    [self doSomething];  // self -> block -> self
};

// ✅ 正确
__weak typeof(self) weakSelf = self;
self.block = ^{
    __strong typeof(weakSelf) strongSelf = weakSelf;
    [strongSelf doSomething];
};
```

### 2. **NSTimer 处理**

```objectivec
// ❌ 错误：Timer 会持有 target
[NSTimer scheduledTimerWithTimeInterval:1.0
                                 target:self
                               selector:@selector(tick)
                               userInfo:nil
                                repeats:YES];

// ✅ 正确：使用 Block API（iOS 10+）
__weak typeof(self) weakSelf = self;
[NSTimer scheduledTimerWithTimeInterval:1.0
                                 repeats:YES
                                   block:^(NSTimer *timer) {
    __strong typeof(weakSelf) strongSelf = weakSelf;
    [strongSelf tick];
}];

// ✅ 或中间对象代理
@interface Proxy : NSObject
@property (weak) id target;
@end

@implementation Proxy
- (void)forwardInvocation:(NSInvocation *)invocation {
    if (self.target) {
        [invocation invokeWithTarget:self.target];
    }
}
@end
```

### 3. **大对象管理**

```objectivec
// 大内存对象及时释放
- (void)processLargeImage {
    @autoreleasepool {
        UIImage *largeImage = [UIImage imageNamed:@"large.jpg"];
        // 处理图片
        [self processImage:largeImage];
        // 离开 @autoreleasepool 时释放
    }
    
    // 确保内存已释放
    // 后续操作...
}

// 使用内存警告处理
- (void)didReceiveMemoryWarning {
    [super didReceiveMemoryWarning];
    
    // 清除缓存
    [self.imageCache removeAllObjects];
    // 释放可重建资源
    [self resetUnusedResources];
}
```

## 六、工具与调试

### **内存检测工具**

```objectivec
// 1. Instruments - Leaks
// 检测内存泄漏

// 2. Zombies
// 检测访问已释放对象

- (void)debugMemory {
    // 获取当前内存使用
    struct task_basic_info info;
    mach_msg_type_number_t size = sizeof(info);
    task_info(mach_task_self(), TASK_BASIC_INFO, (task_info_t)&info, &size);
    NSLog(@"Memory used: %llu MB", info.resident_size / 1024 / 1024);
    
    // 检查对象引用计数

    NSLog(@"Retain count: %ld", rc);  // 警告：只用于调试
}
```

## 七、总结对比表

| 内存区域 | 存储内容 | 管理方式 | 生命周期 | 速度 | 大小限制 |
|---------|---------|---------|---------|-----|---------|
| **代码段** | 指令、常量 | 系统管理 | 程序运行期 | 快 | 固定 |
| **数据段** | 全局/静态变量 | 系统管理 | 程序运行期 | 快 | 固定 |
| **堆区** | 动态对象 | 手动/ARC | 运行时创建/销毁 | 慢 | 大（受物理内存限制） |
| **栈区** | 局部变量、参数 | 自动 | 函数调用期间 | 极快 | 小（~8MB） |

**核心原则**：
- **栈**：小而快，自动管理
- **堆**：大而灵活，需要手动/ARC 管理
- **ARC**：编译器自动插入 retain/release
- **循环引用**：用 `__weak` 打破
- **autoreleasepool**：管理临时对象，避免内存峰值
