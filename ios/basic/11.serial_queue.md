## 一、核心结论

**串行队列之所以所有任务都在同一个线程上执行，是因为 GCD 会将一个线程永久（或长期）绑定给该队列，后续所有任务都复用这个线程。**

```objc
dispatch_queue_t serialQueue = dispatch_queue_create("com.test.serial", DISPATCH_QUEUE_SERIAL);

for (int i = 0; i < 10; i++) {
    dispatch_async(serialQueue, ^{
        NSLog(@"Task %d on thread: %@", i, [NSThread currentThread]);
        // 输出：所有 Task 的线程地址完全相同
    });
}
```

## 二、底层数据结构

### 2.1 串行队列的核心结构

```c
// libdispatch 中的串行队列结构（简化）
struct dispatch_queue_s {
    // 队列类型标识
    uint32_t dq_flags;          // 包含队列类型：SERIAL 或 CONCURRENT
    
    // 队列绑定的线程信息
    pthread_t dq_thread;        // 绑定的工作线程（串行队列专用）
    
    // 线程池相关
    struct dispatch_root_queue_context_s *dq_context;
    
    // 任务队列（FIFO）
    struct dispatch_continuation_s *dq_items_head;
    struct dispatch_continuation_s *dq_items_tail;
    
    // 同步锁
    dispatch_lock_t dq_lock;
};
```

### 2.2 队列类型标志

```c
// 队列类型定义
#define DISPATCH_QUEUE_SERIAL    0x00000001  // 串行标志
#define DISPATCH_QUEUE_CONCURRENT 0x00000002 // 并发标志

// 判断是否为串行队列
static inline bool _dispatch_queue_is_serial(dispatch_queue_t dq) {
    return (dq->dq_flags & DISPATCH_QUEUE_SERIAL) != 0;
}
```

## 三、任务提交到串行队列的完整流程

```c
// 1. 用户调用 dispatch_async
void dispatch_async(dispatch_queue_t queue, dispatch_block_t block) {
    // 包装 block
    struct dispatch_continuation_s *dc = _dispatch_continuation_alloc();
    dc->dc_func = block;
    dc->dc_queue = queue;
    
    // 提交到队列
    _dispatch_async_f(queue, dc, _dispatch_continuation_invoke);
}

// 2. 异步提交的核心函数
void _dispatch_async_f(dispatch_queue_t queue, void *context, dispatch_function_t func) {
    // 加锁
    _dispatch_queue_lock(queue);
    
    // 将任务添加到队列尾部
    _dispatch_queue_add_item(queue, context, func);
    
    // 关键：唤醒队列处理任务
    _dispatch_queue_wakeup(queue);
    
    // 解锁
    _dispatch_queue_unlock(queue);
}
```

### 3.1 唤醒队列的底层实现

```c
void _dispatch_queue_wakeup(dispatch_queue_t dq) {
    // 判断是否为串行队列
    if (_dispatch_queue_is_serial(dq)) {
        // 串行队列：检查是否有活跃线程
        if (dq->dq_thread == NULL) {
            // 第一次唤醒：创建并绑定一个工作线程
            _dispatch_queue_create_worker(dq);
        } else {
            // 已有绑定的线程：唤醒它
            _dispatch_queue_signal_worker(dq);
        }
    } else {
        // 并发队列：从线程池中取线程
        _dispatch_root_queue_wakeup(dq);
    }
}
```

### 3.2 串行队列工作线程的创建与绑定

```c
static void _dispatch_queue_create_worker(dispatch_queue_t dq) {
    // 1. 创建新线程
    pthread_t thread;
    pthread_attr_t attr;
    pthread_attr_init(&attr);
    
    // 设置线程栈大小（子线程默认 512KB）
    pthread_attr_setstacksize(&attr, 512 * 1024);
    
    // 2. 创建线程，入口函数是 _dispatch_worker_thread
    pthread_create(&thread, &attr, _dispatch_worker_thread, dq);
    
    // 3. 将线程绑定到队列（关键！）
    dq->dq_thread = thread;
    
    // 4. 记录线程的 QoS 优先级
    _dispatch_thread_set_qos(thread, dq->dq_qos);
    
    pthread_attr_destroy(&attr);
}
```

### 3.3 工作线程的主循环

```c
// 串行队列的工作线程入口
void* _dispatch_worker_thread(void* context) {
    dispatch_queue_t dq = (dispatch_queue_t)context;
    
    // 设置线程名称
    pthread_setname_np(dq->dq_label);
    
    // 进入无限循环，持续处理队列中的任务
    while (1) {
        // 1. 等待任务到达（信号量）
        _dispatch_queue_wait_for_work(dq);
        
        // 2. 获取队列中的下一个任务
        struct dispatch_continuation_s *dc = _dispatch_queue_dequeue(dq);
        
        if (dc == NULL) {
            // 没有任务，继续等待
            continue;
        }
        
        // 3. 执行任务
        _dispatch_continuation_invoke(dc);
        
        // 4. 检查队列是否还有任务
        if (_dispatch_queue_is_empty(dq)) {
            // 队列为空，线程进入空闲状态
            // 但不会立即销毁，会等待一段时间（复用的关键）
            _dispatch_thread_idle_wait(dq);
        }
        
        // 5. 继续循环，处理下一个任务
    }
    
    return NULL;
}
```

### 3.4 任务出队的 FIFO 实现

```c
// 从队列中取出下一个任务（FIFO）
struct dispatch_continuation_s* _dispatch_queue_dequeue(dispatch_queue_t dq) {
    _dispatch_queue_lock(dq);
    
    // 从头节点取任务
    struct dispatch_continuation_s *dc = dq->dq_items_head;
    
    if (dc != NULL) {
        // 移动头指针
        dq->dq_items_head = dc->dc_next;
        
        if (dq->dq_items_head == NULL) {
            // 队列变空
            dq->dq_items_tail = NULL;
        }
    }
    
    _dispatch_queue_unlock(dq);
    return dc;
}
```

## 四、线程复用的关键机制

### 4.1 空闲线程的等待策略

```c
// 线程执行完任务后不会立即销毁
static void _dispatch_thread_idle_wait(dispatch_queue_t dq) {
    // 设置超时时间（默认 5 秒）
    struct timespec timeout;
    clock_gettime(CLOCK_REALTIME, &timeout);
    timeout.tv_sec += 5;  // 等待 5 秒
    
    // 等待新任务或超时
    int ret = semaphore_timedwait(dq->dq_semaphore, &timeout);
    
    if (ret == 0) {
        // 被新任务唤醒，继续执行
        return;
    } else {
        // 超时：线程空闲太久，销毁线程
        dq->dq_thread = NULL;
        pthread_exit(NULL);
    }
}
```

### 4.2 为什么同一个线程？

```c
// 关键点：串行队列的 dq_thread 指针是固定不变的
// 第一次创建线程后，dq_thread 指向这个线程
// 后续所有任务都是唤醒这个线程，而不是创建新线程

// 第一次任务
dispatch_async(serialQueue, task1);
// → 发现 dq_thread == NULL → 创建线程 T1 → dq_thread = T1 → T1 执行 task1

// 第二次任务
dispatch_async(serialQueue, task2);
// → 发现 dq_thread != NULL → 唤醒 T1 → T1 执行 task2（同一个线程）

// 第三次任务
dispatch_async(serialQueue, task3);
// → 唤醒 T1 → T1 执行 task3（还是同一个线程）
```

## 五、与并发队列的对比

| 特性 | 串行队列 (SERIAL) | 并发队列 (CONCURRENT) |
|------|-------------------|----------------------|
| **线程绑定** | 固定绑定一个线程 | 不固定，从线程池取 |
| **线程数量** | 1 个 | 多个（动态） |
| **队列类型标志** | `DISPATCH_QUEUE_SERIAL` | `DISPATCH_QUEUE_CONCURRENT` |
| **唤醒逻辑** | `_dispatch_queue_create_worker` | `_dispatch_root_queue_wakeup` |
| **任务执行** | 顺序执行 | 可能并发执行 |

```c
// 并发队列的唤醒逻辑（对比）
void _dispatch_root_queue_wakeup(dispatch_queue_t dq) {
    // 从线程池获取线程
    int worker_count = dq->dq_thread_pool_size;
    
    if (worker_count < dq->dq_max_threads) {
        // 可以创建新线程
        pthread_create(&thread, NULL, _dispatch_worker_thread, dq);
        dq->dq_thread_pool_size++;
    } else {
        // 达到上限，唤醒已有线程
        semaphore_signal(dq->dq_semaphore);
    }
}
```

## 六、验证实验

### 实验1：串行队列确实固定线程

```objc
dispatch_queue_t serialQueue = dispatch_queue_create("test.serial", DISPATCH_QUEUE_SERIAL);

dispatch_async(serialQueue, ^{
    NSLog(@"Task1 thread: %p", [NSThread currentThread]);
});

dispatch_async(serialQueue, ^{
    NSLog(@"Task2 thread: %p", [NSThread currentThread]);
    // 输出：与 Task1 相同的线程地址
});

dispatch_async(serialQueue, ^{
    [NSThread sleepForTimeInterval:3];
    NSLog(@"Task3 thread: %p", [NSThread currentThread]);
    // 还是同一个线程
});
```

### 实验2：队列销毁后线程回收

```objc
// 创建串行队列
dispatch_queue_t queue = dispatch_queue_create("test", DISPATCH_QUEUE_SERIAL);

dispatch_async(queue, ^{
    NSLog(@"Task on thread: %@", [NSThread currentThread]);
});

// 5 秒后队列被释放，线程也会被回收
queue = nil;
```

## 七、总结

| 问题 | 答案 |
|------|------|
| **为什么在同一个线程？** | 串行队列会将一个线程永久绑定给自己 |
| **什么时候绑定线程？** | 第一次提交任务时 |
| **如何保证串行？** | 同一个线程顺序处理队列中的任务 |
| **线程会销毁吗？** | 空闲约 5 秒后销毁 |
| **与并发队列的区别？** | 并发队列从线程池取线程，不固定 |

**一句话总结**：串行队列在第一次接收任务时创建一个工作线程并保存其引用，后续所有任务都通过唤醒这个线程来执行，从而保证了任务在同一个线程上顺序执行。
