# Swift 并发三部曲：TaskGroup、Actor、AsyncSequence

## 一、TaskGroup：结构化并发

### 1. 什么是 TaskGroup？

TaskGroup 让你可以**动态创建一组并发子任务**，并等待它们全部完成。这是结构化并发的核心。

### 2. 基本用法

```swift
// 场景：并发下载多个图片
func downloadImages(urls: [URL]) async throws -> [UIImage] {
    try await withThrowingTaskGroup(of: UIImage.self) { group in
        // 1. 添加任务
        for url in urls {
            group.addTask {
                return try await downloadImage(from: url)
            }
        }

        // 2. 收集结果
        var images: [UIImage] = []
        for try await image in group {
            images.append(image)
        }
        return images
    }
    // 3. 自动等待所有任务完成
}
```

### 3. 动态控制任务数量

```swift
// 场景：批量处理，限制并发数
func processBatch(items: [Item], maxConcurrent: Int) async throws -> [Result] {
    try await withThrowingTaskGroup(of: Result.self) { group in
        var results: [Result] = []
        var iterator = items.makeIterator()

        // 初始添加 maxConcurrent 个任务
        for _ in 0..<maxConcurrent {
            guard let item = iterator.next() else { break }
            group.addTask { try await process(item) }
        }

        // 动态补充任务
        while let result = try await group.next() {
            results.append(result)
            if let nextItem = iterator.next() {
                group.addTask { try await process(nextItem) }
            }
        }

        return results
    }
}
```

### 4. 任务取消传播

```swift
// 场景：超时控制
func fetchWithTimeout(url: URL, timeout: Duration) async throws -> Data {
    try await withThrowingTaskGroup(of: Data.self) { group in
        // 添加实际任务
        group.addTask {
            return try await downloadData(from: url)
        }

        // 添加超时任务
        group.addTask {
            try await Task.sleep(for: timeout)
            throw TimeoutError()
        }

        // 谁先完成就取谁的结果
        let result = try await group.next()!

        // 取消另一个任务（重要！）
        group.cancelAll()

        return result
    }
}
```

### 5. 任务组的值传递

```swift
// 场景：Map-Reduce 模式
func parallelMap<T, U>(_ array: [T],
                       transform: @escaping (T) async -> U) async -> [U] {
    await withTaskGroup(of: (Int, U).self) { group in
        // 并发处理，保留原始顺序
        for (index, element) in array.enumerated() {
            group.addTask {
                let result = await transform(element)
                return (index, result)
            }
        }

        // 收集并按顺序重组
        var results = [(Int, U)]()
        for await result in group {
            results.append(result)
        }
        return results.sorted { $0.0 < $1.0 }.map { $0.1 }
    }
}

// 使用
let numbers = Array(1...100)
let squares = await parallelMap(numbers) { number in
    return number * number
}
```

## 二、Actor：数据隔离

### 1. 什么是 Actor？

Actor 是**引用类型**，保护其状态不被多个任务同时访问，避免数据竞争。

```swift
actor BankAccount {
    private var balance: Double = 0

    func deposit(_ amount: Double) {
        balance += amount  // 安全：Actor 隔离
    }

    func withdraw(_ amount: Double) throws {
        guard balance >= amount else {
            throw BankError.insufficientFunds
        }
        balance -= amount
    }

    func getBalance() async -> Double {
        return balance  // 需要 async，因为可能等待
    }
}

// 使用
let account = BankAccount()
await account.deposit(100)  // 需要 await
```

### 2. Actor 重入问题

```swift
actor DataCache {
    private var cache: [String: Data] = [:]
    private var loadingTasks: [String: Task<Data, Error>] = [:]

    func getData(key: String) async throws -> Data {
        // 检查缓存
        if let cached = cache[key] {
            return cached
        }

        // 检查是否已有加载任务
        if let existingTask = loadingTasks[key] {
            return try await existingTask.value
        }

        // 创建新任务
        let task = Task { () -> Data in
            let data = try await fetchData(key: key)
            await self.updateCache(key: key, data: data)
            return data
        }

        loadingTasks[key] = task
        defer { loadingTasks[key] = nil }

        return try await task.value
    }

    private func updateCache(key: String, data: Data) {
        cache[key] = data  // 在 actor 内部，安全
    }
}
```

### 3. 可发送性检查

```swift
// 非发送类型
class NonSendable {
    var value: Int = 0

actor MyActor {
    func badExample() async {
        let local = NonSendable()  // ⚠️ 警告：可能被并发访问

        Task.detached {

        }
    }

    func goodExample() async {
        let local = SendableWrapper(value: 10)  // ✅ 可发送类型

        Task.detached {
            print(local.value)
        }
    }
}

// 定义可发送类型
struct SendableWrapper: Sendable {
    let value: Int
}
```

## 三、AsyncSequence：异步序列

### 1. 什么是 AsyncSequence？

AsyncSequence 让你可以**异步遍历序列**，每次获取下一个值时可能挂起。

```swift
// 基本使用
let numbers = [1, 2, 3, 4, 5].async
for await number in numbers {
    print(number)  // 同步遍历，但支持挂起
}

// 自定义异步序列
struct Counter: AsyncSequence {
    typealias Element = Int

    let limit: Int

    struct AsyncIterator: AsyncIteratorProtocol {
        let limit: Int
        var current = 0

        mutating func next() async -> Int? {
            guard current < limit else { return nil }
            try? await Task.sleep(nanoseconds: 1_000_000_000)  // 模拟延迟
            current += 1
            return current
        }
    }

    func makeAsyncIterator() -> AsyncIterator {
        return AsyncIterator(limit: limit)
    }
}

// 使用
for await number in Counter(limit: 5) {
    print(number)  // 每秒输出一个数字
}
```

### 2. AsyncStream 创建异步序列

```swift
// 方式1：使用 AsyncStream
func listenToNotifications() -> AsyncStream<Notification> {
    AsyncStream { continuation in
        let observer = NotificationCenter.default.addObserver(
            forName: .someNotification,
            object: nil,
            queue: nil
        ) { notification in
            continuation.yield(notification)
        }

        continuation.onTermination = { _ in
            NotificationCenter.default.removeObserver(observer)
        }
    }
}

// 使用
for await notification in listenToNotifications() {
    print("收到通知: \(notification)")
}
```

### 3. AsyncThrowingStream 带错误处理

```swift
// 场景：分页加载网络数据
func searchPages(query: String) -> AsyncThrowingStream<[SearchResult], Error> {
    AsyncThrowingStream { continuation in
        Task {
            var page = 1
            var hasMore = true

            do {
                while hasMore {

                    continuation.yield(results)

                    hasMore = results.count == pageSize
                    page += 1

                    if !hasMore {
                        continuation.finish()
                    }
                }
            } catch {
                continuation.finish(throwing: error)
            }
        }
    }

// 使用
```

### 4. AsyncSequence 的操作符

```swift
// 使用 Combine-like 操作符
import AsyncAlgorithms  // 需要添加依赖

let sequence = AsyncStream { continuation in
    for i in 1...10 {
        continuation.yield(i)
        try? await Task.sleep(nanoseconds: 100_000_000)
    }
    continuation.finish()
}

// 链式操作
let transformed = sequence
    .filter { $0 % 2 == 0 }           // 只取偶数
    .map { "Number: \($0)" }          // 转换
    .prefix(3)                         // 只取前3个
    .debounce(for: .seconds(0.5))     // 防抖

for await value in transformed {
    print(value)  // "Number: 2", "Number: 4", "Number: 6"
}
```

### 5. 实战：实时搜索

```swift
class SearchService {
    func search(query: AsyncStream<String>) -> AsyncStream<[SearchResult]> {
        var currentTask: Task<Void, Never>?

        return AsyncStream { continuation in
            Task {
                for await text in query {
                    // 取消之前的搜索
                    currentTask?.cancel()

                    currentTask = Task {
                        // 防抖
                        try? await Task.sleep(nanoseconds: 300_000_000)

                        guard !Task.isCancelled else { return }

                        if text.count >= 2 {
                            let results = await performSearch(text)
                            continuation.yield(results)
                        } else {
                            continuation.yield([])
                        }
                    }

                    await currentTask?.value
                }
                continuation.finish()
            }
        }
    }

    private func performSearch(_ query: String) async -> [SearchResult] {
        // 模拟网络请求
        try? await Task.sleep(nanoseconds: 500_000_000)
        return [SearchResult(title: "Result for \(query)")]
    }
}

// 使用
let queryStream = AsyncStream(String.self) { continuation in
    // 模拟用户输入
    continuation.yield("S")
    continuation.yield("Sw")
    continuation.yield("Swi")
    continuation.yield("Swif")
    continuation.yield("Swift")
    continuation.finish()
}

let searchService = SearchService()
for await results in searchService.search(query: queryStream) {
    print("搜索结果: \(results)")
}
```

## 四、三者结合：完整示例

```swift
// 场景：并发下载，带缓存，分批返回

// 1. 定义 Actor 管理缓存
actor ImageCache {
    private var cache: [String: UIImage] = [:]
    private var tasks: [String: Task<UIImage, Error>] = [:]

    func getImage(url: String) async throws -> UIImage {
        // 检查缓存
        if let cached = cache[url] {
            return cached
        }

        // 复用已有任务
        if let existingTask = tasks[url] {
            return try await existingTask.value
        }

        // 创建新任务
        let task = Task { () -> UIImage in
            let image = try await downloadImage(url: url)
            await self.storeInCache(url: url, image: image)
            return image
        }

        tasks[url] = task
        defer { tasks[url] = nil }

        return try await task.value
    }

    private func storeInCache(url: String, image: UIImage) {
        cache[url] = image
    }
}

// 2. 异步序列，边下载边返回
func downloadImages(urls: [String]) -> AsyncThrowingStream<UIImage, Error> {
    AsyncThrowingStream { continuation in
        Task {
            let cache = ImageCache()

            await withThrowingTaskGroup(of: UIImage.self) { group in
                for url in urls {
                    group.addTask {
                        return try await cache.getImage(url: url)
                    }
                }

                do {
                    for try await image in group {
                        // 每下载完一个就立即返回
                        continuation.yield(image)
                    }
                    continuation.finish()
                } catch {
                    continuation.finish(throwing: error)
                }
            }
        }
    }
}

// 3. 使用
Task {
    let urls = ["url1", "url2", "url3", "url4", "url5"]

    do {
        for try await image in downloadImages(urls: urls) {
            // 图片一个接一个显示，不用等全部下载完
            await MainActor.run {
                displayImage(image)
            }
        }
        print("全部下载完成")
    } catch {
        print("下载失败: \(error)")
    }
}
```

## 五、选择指南

| 场景                 | 使用方案                  |
| -------------------- | ------------------------- |
| 并发执行多个独立任务 | TaskGroup                 |
| 等待所有任务完成     | TaskGroup                 |
| 保护共享数据状态     | Actor                     |
| 防止数据竞争         | Actor                     |
| 逐步产生数据         | AsyncSequence             |
| 实时数据流           | AsyncSequence             |
| 缓存管理             | Actor + TaskGroup         |
| 批量处理             | TaskGroup + AsyncSequence |

## 六、总结

**TaskGroup**：

- 结构化并发，自动管理子任务生命周期
- 支持动态添加任务
- 取消传播自动处理

**Actor**：

- 编译器强制执行数据隔离
- 避免锁和手动同步
- 支持重入（需注意）

**AsyncSequence**：

- 异步遍历，按需生产数据
- 支持背压控制
- 可与 Combine 互操作

这三者构成了 Swift 并发编程的基石，合理组合可以写出既安全又高效的异步代码。

## 七、Task 与线程的关系

这是一个非常核心的问题！理解 Task 和线程的关系，是掌握 Swift 并发的关键。

### 1. 核心结论

**Task 不绑定线程，线程是执行 Task 的资源。**

```swift
// Task 可以在不同线程上执行
Task {
    print("开始: \(Thread.current)")
    await Task.sleep(1_000_000_000)
    print("恢复: \(Thread.current)")  // 可能在不同线程！
}

// 输出示例：
// 开始: <NSThread: 0x6000...> number = 5
// 恢复: <NSThread: 0x6000...> number = 7  ← 线程变了！
```

### 2. Task 与线程的对比

| 特性         | Task                 | 线程                  |
| ------------ | -------------------- | --------------------- |
| **抽象级别** | 高层抽象（任务单元） | 底层系统资源          |
| **创建开销** | 极小（约 1KB）       | 较大（约 1MB 栈空间） |
| **数量限制** | 可创建数十万个       | 受限（通常几百）      |
| **调度方式** | Swift 运行时协作调度 | 操作系统抢占式调度    |
| **绑定关系** | 不绑定线程           | 固定内核线程          |
| **挂起恢复** | 语言级支持           | 系统级上下文切换      |

### 3. 线程池模型

Swift 运行时维护一个**协作式线程池**：

```swift
// Swift 运行时的线程池（概念模型）
actor ThreadPool {
    private var threads: [Thread] = []
    private var taskQueue: [Task] = []

    func schedule(_ task: Task) {
        if let idleThread = findIdleThread() {
            // 绑定任务到空闲线程
            idleThread.run(task)
        } else {
            // 入队等待
            taskQueue.append(task)
        }
    }
}
```

**线程复用示例**：

```swift
// 创建 1000 个 Task
for i in 0..<1000 {
    Task {
        print("Task \(i) 运行在线程: \(Thread.current.number)")
        await Task.sleep(1_000_000)
    }
}

// 虽然创建了 1000 个 Task，但只用了少数几个线程
// 输出示例：
// Task 0 运行在线程: 5
// Task 1 运行在线程: 6
// Task 2 运行在线程: 7
// Task 3 运行在线程: 5  ← 线程 5 被复用
// Task 4 运行在线程: 6
// ...
```

### 4. 协作式调度 vs 抢占式调度

#### 线程（抢占式）

```swift
// 操作系统决定何时切换线程
// 你无法控制

Thread {
    while true {
        // 操作系统可能随时暂停这个线程
        // 切换到其他线程
        print("运行中...")
    }
}
```

#### Task（协作式）

```swift
// Task 只在 `await` 时主动让出控制权

Task {
    print("开始")
    await Task.sleep(1_000)  // ← 主动挂起，让出线程
    print("恢复")             // 后面可能在不同线程
}

// 没有 await 的 Task 会一直占用线程
Task {
    for i in 0..<1000000 {
        // ❌ 没有 await，不会挂起
        // 会一直占用线程直到完成
        compute(i)
    }
}
```

### 5. 线程切换示例

#### 示例1：挂起时可能切换线程

```swift
func demonstrateThreadSwitching() async {
    print("1️⃣ 主线程: \(Thread.current)")

    await Task.yield()  // 主动让出

    print("2️⃣ yield 后: \(Thread.current)")

    await Task.sleep(1_000_000_000)  // 等待1秒

    print("3️⃣ sleep 后: \(Thread.current)")
}

Task {
    await demonstrateThreadSwitching()
}

// 可能输出：
// 1️⃣ 主线程: <NSThread: 0x...> number = 1
// 2️⃣ yield 后: <NSThread: 0x...> number = 5  ← 线程切换了
// 3️⃣ sleep 后: <NSThread: 0x...> number = 7  ← 又切换了
```

#### 示例2：Actor 确保串行但不保证线程

```swift
actor Counter {
    var value = 0

    func increment() async {
        print("当前线程: \(Thread.current.number)")
        value += 1
        await Task.yield()
        print("恢复线程: \(Thread.current.number)")
    }
}

let counter = Counter()

Task {
    await counter.increment()
}
Task {
    await counter.increment()
}

// 虽然两个任务顺序执行（不会同时），但可能在不同线程
// 输出：
// 当前线程: 5
// 恢复线程: 7   ← 同一个 Actor 的方法，恢复时线程变了
// 当前线程: 5   ← 第二个任务开始
// 恢复线程: 6
```

### 6. 线程绑定的特殊情况

#### 1. MainActor 绑定主线程

```swift
@MainActor
class ViewModel {
    func updateUI() {
        // 保证在主线程执行
        assert(Thread.isMainThread)
    }
}

Task { @MainActor in
    // 这个闭包总是运行在主线程
    print(Thread.isMainThread)  // true
}
```

#### 2. 自定义 Executor 绑定特定线程

```swift
// 可以创建绑定到特定线程的 Executor
final class ThreadBoundExecutor: SerialExecutor {
    private let thread: Thread

    init(thread: Thread) {
        self.thread = thread
    }

    func enqueue(_ job: UnownedJob) {
        performOnThread(thread) {
            job.runSynchronously()
        }
    }
}

// 使用自定义 Executor
actor MyActor {
    private let executor = ThreadBoundExecutor(thread: myThread)

    nonisolated var unownedExecutor: UnownedSerialExecutor {
        executor.asUnownedSerialExecutor()
    }
}
```

## 七、性能对比

### 线程创建开销

```swift
// 创建线程（昂贵）
for _ in 0..<1000 {
    Thread {
        // 每个线程占用 ~1MB 栈空间
        // 创建耗时 ~微秒级别
    }.start()
}
// 1000 个线程 = 约 1GB 内存，可能崩溃！

// 创建 Task（轻量）
for _ in 0..<100_000 {
    Task {
        // 每个 Task 只占用 ~1KB
        // 创建耗时 ~纳秒级别
    }
}
// 10 万个 Task 完全没问题
```

### 上下文切换开销

```swift
// 线程切换：系统级，昂贵（微秒级别）
// 需要保存所有寄存器、切换内存映射等

// Task 挂起：语言级，极快（纳秒级别）
// 只保存必要状态到堆上
await Task.yield()  // 极快的协作式切换
```

## 八、实际应用建议

### 1. 避免假设线程不变

```swift
// ❌ 错误：假设线程不变
func badPattern() async {
    let thread = Thread.current
    await someAsync()
    assert(thread == Thread.current)  // 可能失败！
}

// ✅ 正确：不依赖线程
func goodPattern() async {
    let id = UUID()  // 使用逻辑标识，不是线程
    await someAsync()
    // 继续工作，不关心在哪个线程
}
```

### 2. 需要特定线程时使用 MainActor

```swift
// UI 更新必须在主线程
@MainActor
func updateUI() {
    // 保证在主线程
    self.label.text = "Updated"
}
```

### 3. CPU 密集型任务避免阻塞

```swift
// ❌ 不好：长时间没有挂起点
func badHeavyWork() async {
    for i in 0..<1_000_000 {
        compute(i)  // 没有 await，会阻塞线程
    }
}

// ✅ 好：定期让出
func goodHeavyWork() async {
    for i in 0..<1_000_000 {
        compute(i)
        if i % 1000 == 0 {
            await Task.yield()  // 给其他任务机会
        }
    }
}
```

## 十、总结

| 概念       | 关系                             |
| ---------- | -------------------------------- |
| **Task**   | 逻辑工作单元，轻量，可挂起恢复   |
| **Thread** | 物理执行资源，重量，操作系统管理 |
| **绑定**   | Task 不绑定线程，可在线程间迁移  |
| **调度**   | Task 协作式，Thread 抢占式       |
| **数量**   | 可大量创建 Task，线程受系统限制  |
| **切换**   | Task 切换极快，线程切换较慢      |

**核心理解**：

- Task 是"要做的事"
- 线程是"做事的人"
- 人可以切换做不同的事（Task 切换线程）
- 多个人可以协作做很多事（线程池）
- 重要的事指定专人做（MainActor 绑定主线程）

**记忆口诀**：

- Task 是任务，线程是工人
- 工人可换任务，任务可换工人
- 创建轻量成本低，数量众多不担心
- 挂起恢复线程换，高效并发是根本

这是一个很好的跨平台对比问题。Swift 的 `Task` 和 Kotlin 协程在核心理念上非常相似（都基于结构化并发和协作式调度），但在实现哲学和语言特性上有着根本性的差异。

### 一句话核心区别

> **Kotlin 协程是“侵入式”的，它通过“结构化并发”在编译期严格管控生命周期；而 Swift 的 Task 虽然也支持结构化并发，但同时也给予开发者更多基于 Task 对象的“非结构化”自由。**

为了帮你更直观地理解，我把它们的核心区别整理成了下面这张表：

| 维度             | 🍎 Swift Task                                                                       | 🤖 Kotlin Coroutines                                                             |
| :--------------- | :---------------------------------------------------------------------------------- | :------------------------------------------------------------------------------- |
| **生命周期管理** | **无默认 Scope**。Task 默认不绑定生命周期，需要开发者手动持有或依赖 `task` 修饰符。 | **默认 Scope**。通过 `viewModelScope`、`lifecycleScope` 绑定生命周期，自动取消。 |
| **线程调度**     | **无直接控制权**。依赖全局协作线程池，不支持指定调度器，强调“不关心线程”。          | **精细控制**。通过 `Dispatchers.IO`、`Dispatchers.Default` 等指定具体线程池。    |
| **数据安全**     | **编译器强制**。Swift 6 严格检查 `Sendable`，不符合协议的类无法跨任务传递。         | **运行时/约定**。不强制检查，主要靠开发者遵循线程安全规范（如使用 `Mutex`）。    |
| **异步流**       | **AsyncStream**。语言原生支持，可与 `for await` 无缝集成。                          | **Flow**。独立库（非 Kotlin 语言内置），功能强大但需要额外引入。                 |
| **取消机制**     | **协作式**。需检查 `Task.isCancelled`，运行时不强制立即停止。                       | **协作式**。需检查 `isActive` 或调用 `ensureActive()`，行为类似。                |
| **核心关键词**   | `async`、`await`、`Task`、`Actor`                                                   | `suspend`、`launch`、`async`、`Dispatcher`                                       |

### 🔍 深度剖析差异本质

#### 1. 生命周期管理

- **Kotlin：Scope 无处不在**
  协程的设计与 Android 生命周期深度绑定。`viewModelScope` 或 `lifecycleScope` 会在页面销毁时自动取消所有子任务。这是一种“结构化并发”的强制实践，可以有效防止内存泄漏。

- **Swift：Task 需要显式管理**
  创建一个 `Task` 就像是“射出一颗子弹”，它默认并不自动绑定到当前页面。如果希望在页面消失时取消任务，要么使用 `.task` 修饰符（iOS 15+），要么手动持有 Task 并在 `onDisappear` 中调用 `cancel()`。

#### 2. 线程调度

- **Swift：告别“线程思维”**
  Swift 并发强调的是 **“协作式线程池”** 。设计者希望你**不再**关心代码具体跑在哪个线程上，只需要关注函数的执行顺序和 Actor 隔离。因此，Swift 没有像 `Dispatchers.IO` 这样的 API。

- **Kotlin：上下文切换是常态**
  协程的 `withContext(Dispatchers.IO)` 是标准写法。开发者需要清晰地知道：网络请求切 IO，计算结果切 Default，UI 更新切 Main。这是一种更传统、显式的控制方式。

#### 3. 数据安全与跨平台（KMM 痛点）

这是 KMP 开发中最容易被卡住的地方。

- **Swift 6 的严格检查**：Swift 6 引入了 `Sendable` 协议。如果一个对象要在多个并发任务间传递，必须标记为安全（不可变或锁保护）。
- **Kotlin 的默认生成**：Kotlin 默认生成的 `data class` 是可变的，不符合 Swift 的 `Sendable` 要求。这导致在 KMP 项目中，Kotlin 返回的对象**无法直接**在 Swift 并发环境中使用，编译器会报错。

### 💡 开发者应该如何选择？

你的选择其实取决于应用场景：

1.  **如果你是纯原生 iOS / SwiftUI 开发**：
    优先拥抱 **Swift Task**。配合 Swift 6 的严格并发检查和 `@MainActor`，它们是最安全、最符合苹果生态演进方向的选择。

2.  **如果你是纯原生 Android / Kotlin 开发**：
    毫无疑问选择 **Kotlin Coroutines**。它与 Jetpack 组件（ViewModel、Room、Retrofit）的集成度最高，生态最完善。

3.  **如果你在做 KMP（Kotlin Multiplatform）跨平台开发**：
    这里有个“最佳实践”的金法则：**不要让共享层直接暴露协程 API**。
    - **策略**：在共享模块（`commonMain`）中，只使用普通的 `suspend` 函数（不指定 `Dispatcher`）。
    - **原因**：这样可以在 iOS 端利用 `SKIE` 等工具将其转换为原生的 Swift `async` 函数，避免触犯 Swift 的并发安全红线。
